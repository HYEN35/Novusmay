CSS 작업 규칙

기본규칙

CSS 컨벤션은 NHN Coding Conventions_for_Markup Languages을 기준으로 한다.

CSS 작성은 공통 영역 > 마크업 구조 순서로 한다.

한 줄 쓰기로 작성한다.

마지막 세미콜론(;)은 생략한다.

선택자와 중괄호 사이의 공백은 사용하지 않는다.

클래스명이 아닌 태그로 CSS를 선언하는 경우 자식선택자(>)를 사용한다.

CSS 선언 순서를 유의하며 작성한다.

플러그인 CSS는 vendor 폴더에 저장한다.

CSS 선언 순서

순서 의미 대표되는 속성(그룹) 관련 속성 기타
1 표시 display block, inline-block, none -
2 넘침 overflow hidden, auto, scroll -
3 흐림 float left, right, unset -
4 위치 position relative, absolute, fixed, stactic, top, right, bottom, left 위치 값 선언할 때 순서 유의하여 작성(top, right, bottom, left)
5 크기 width, height, min-width, min-height - -
6 간격 margin, padding top, right, bottom, left 축약형으로 사용 가능/순서 유의
7 배경 background background, background-color, background-size, background-position 등 background 축약형으로 사용 가능
8 글자 font color, font-size, font-weight, text-align, line-height 등 축약형으로 사용 가능/순서 유의
9 테두리 border border, border-radius, box-size 축약형으로 사용 가능/순서 유의/4면 보더 사용시 꼭 box-size 선언
10 동작 animation transform 등 -
11 배치 z-index - dimmed의 z-index를 1500으로 설정하고 이를 기준으로 상황에 맞게 선언
12 기타 content 등 - 위에 언급되지 않은 속성들은 동작 속성 이후에 선언하며, 기타 내 속성의 선언 순서는 없다. 자유롭게 작성 가능하다.

공통 CSS

공통 CSS

reset.css font.css

공통 CSS는 솔루션과 홈페이지 모두 동일한 파일을 사용한다.

common.css는 솔루션 작업할 때 사용한다.

웹 관리자, 사용자, 에이전트 등에서 공통으로 사용되는 css를 common.css에 작성한다. ex) popup, pagenation, button 등

reset.css는 수정하지 않는다.

font.css는 솔루션별 해당하는 폰트만 사용한다.

공통 CSS 순서

vendor css > reset.css > font.css > common.css

벤더프리픽스(Vendor Prefix)

익스플로러 구현때문에 벤더프리픽스를 사용한다.

transfome, placeholder에 CSS를 선언하는 경우에만 사용한다.

선언 순서

webkt -> ms -> moz -> o -> 일반 o(오페라)는 생략 가능
