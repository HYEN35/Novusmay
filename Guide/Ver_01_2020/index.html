HTML 작업규칙

문서형 선언

DTD:Document Type Definition DTD 선언은 기본 html5에서 제공하는 것을 기본으로 합니다.

<!DOCTYPE html>
언어 지정

언어 지정은 기본 한글로 합니다.

<html lang="ko">
문자셋(charset) 지정

문자셋 지정은 기본 utf-8로 합니다.

<meta charset="utf-8">
head 구조

head의 순서는 title, meta, meta og, link 순서로 합니다. script는 head에 넣지 않습니다. css는 vandor css, font, reset, style 순서로 합니다.

파일 경로

파일 경로는 상대경로로 표기합니다. 같은 위치에 있는 경우에도 './'는 생략하지 않습니다. 메뉴 링크에만 './'를 생략합니다.

코드 들여쓰기

코드의 가독성을 위해 코드 들여쓰기 규칙을 준수합니다. 스크립트 작성시에도 들여쓰기 규칙을 준수합니다. html의 자식 요소인 head, body의 경우에는 들여쓰기를 하지 않습니다. head안에 들어가는 요소들은 1탭 들여쓰기로 작성합니다.

빈줄

객체를 감싸고 있는 그룹을 구분하기 위해 1줄 빈칸은 허용합니다. 빈칸은 1줄을 초과하지 않으며 주석 다음줄은 빈칸으로 두지 않습니다.

주석

마크업에서의 주석은 개발 영역에 설명이 필요한 경우에만 사용합니다. 주석을 달 경우에는 말머리에 [D]를 씁니다. 구조 설명을 위한 주석은 피합니다. 스크립트에서 주석은 해당 함수를 설명하기 위한 경우에만 사용합니다.

버튼의 사용 범위 : a태그와 button 태그의 사용

버튼에는 button 태그를 사용합니다. 앵커 이동, 파일을 다운로드 하는 기능의 버튼인 경우에만 a태그를 사용합니다. input type을 사용해 버튼을 만드는 것은 피합니다.

i태그 활용

i태그는 기본적으로 폰트 기울기 태그이나 기울기 기능을 리셋 후 아이콘 또는 개발에서 데이터값이 들어가는 태그에 사용합니다. 태그 중 가장 작은 단위의 태그

#none

href, value등에서 공란으로 처리해야하는 경우에는 '#none'으로 처리합니다.

Form 사용

form은 마크업단에서 사용하지 않습니다.

페이지 작업 순서

페이지 작업 순서는 웹 흐름의 순서대로 합니다. ex) 로그인 > 메인 > 메뉴 순서대로

W3C 검사

마크업 작업이 완료 된 후(완료처리 하기 전), W3C 검사는 필수로 합니다. 한 페이지 작업 완료 후 W3C 검사를 합니다. W3C 검사에서 에러(빨간경고) 처리를 한 후에 작업 완료 처리를 합니다.

프로젝트를 진행하기 전 index.html/html_list.html(이하 index.html)를 작성합니다. index.html에는 프로젝트명, 작업자, 작업일, 홈페이지 스타일(반응형 또는 웹/태블릿/앱)을 작성합니다. 기존 index.html 파일을 활용하여 작성합니다. index.html은 서비스와 홈페이지에서 파일명이 각각 다릅니다. 서비스 일 때 : index.html 홈페이지 일 때 : html_list.html

index.html 파일에 웹, 앱, 에이전트 모든 페이지를 리스트업 합니다.

파일명 : html_list.html

메인페이지 파일명은 index.html로 작업합니다.

CSS 작업규칙

기본규칙

CSS 컨벤션은 'NHN Coding Conventions_for_Markup Languages'를 기준으로 합니다. CSS 작성 순서는 마크업 구조의 순서로 작성합니다. 공통 CSS를 가장 먼저 작성합니다. 미디어 쿼리는 가장 마지막에 작성하며, 큰 브라우저의 크기를 먼저 작성합니다. 미디어 쿼리는 max-width값으로 작성합니다. CSS는 모두 한줄쓰기로 작성합니다. ex) .download{width:100px;background-color:#fff;color:#07235e;border:1px solid #07235e} 마지막 ;는 생략합니다. ex) .slt_bx{float:left;margin-right:8px} 선택자와 중괄호의 사이에 공백을 사용하지 않는다. ex) .page_btn{border-top:1px solid #ebebeb} 클래스명을 선언하지 않고 태그를 사용하는 경우에는 꺽쇠(>)를 사용합니다. ex) .item>dd .btn{float:left} CSS 선언 순서는 아래 표를 참고 CSS는 한 파일당 5000줄 이상 넘어가지 않습니다.

css파일 네이밍 규칙

플러그인 css의 경우에는 vendor 폴더 안에 저장합니다. [서비스의 경우] web.css, user.css, mobile.css, agent.css, admin.css로 나눠서 작업합니다. [서비스의 경우] 고객사 커스텀의 경우에는 '_고객사명'을 추가합니다. ex) 경기대학교 커스텀 : web_kgu.css, user_kgu.css [홈페이지의 경우] style.css로 나눠서 작업합니다.

아이디/클래스 네이밍 규칙

ID의 사용은 피합니다. 불가피하게 사용해야하는 경우(input-label)를 제외하고는 ID는 사용하지 않습니다. 클래스로 작업해주세요. 모두 소문자로 작성합니다. 두 단어 이상의 조합인 경우에는 언더바(_)를 사용합니다. ex) css_setting (O) / CssSetting (X) / cssSetting (X) 세 단어 이상의 조합은 피합니다. ex) today_select_bx (O) / today_select_bx_item (X) 약어 사용은 피합니다. 다만, 모두가 공통으로 사용하고 있는 약어들은 사용 가능합니다. ex) btn, cnt, tit, txt, img, bx 등 숫자를 붙이는 경우에는 _0N으로 작성합니다. ex) item_01 (O) / today_select_bx_01 (O) / item1 (X) / item_1 (X)

image 삽입 규칙 : img 태그와 css background 사용

이미지는 background로 넣습니다. 로고, 프로필이미지와 같이 커스텀, 개발이 필요한 이미지만 img태그를 사용하여 마크업에 넣습니다.

크로스 브라우징

크로스 브라우징은 익스플로러 9버전까지 확인합니다. css에서 webkit 사용은 transform, placeholder만 사용합니다. 웹킷작성 순서는 -webkit- > -ms- > -moz- > 일반 순으로 작성합니다.

ex) .logo_wrap{position:absolute;top:50%;left:50%;-webkit-transform:translate(-50%,-50%);-ms-transform:translate(-50%,-50%);transform:translate(-50%,-50%)}

클리어픽스 사용

클리어픽스는 float한 요소가 깨지는 것을 방지합니다. float:left 또는 float:right를 선언 한 후, 부모요소에 클리어픽스를 선언합니다. 클리어픽스는 가상선택자 after를 사용하여 작성합니다.

작성예시) div:after{display:block;clear:both;content:""}

가상선택자 :before, :after 사용

가상선택자 after는 클리어픽스를 사용할때만 사용합니다. 그 외의 경우에는 사용을 피합니다. 클리어픽스를 제외한 가상선택자는 before를 사용합니다.

dimmed와 z-index의 사용

dimmed의 z-index는 무조건 1500으로 합니다. dimmed의 z-index를 기준으로 100단위로 가감해서 사용합니다.

영역을 나타내는 클래스와 그 크기

영역을 정할 때 사용하는 클래스는 _area, _section, _cnt, _bx, _wrap을 사용합니다. 클래스의 크기는 _area > _section > _cnt > _bx > _wrap으로 합니다. _wrap이 가장 작은 영역입니다.

서비스에서 버튼 작업

서비스의 경우 다국어 작업을 고려하여 작업을 해야하기때문에 width 고정값을 사용하지 않습니다.

*버튼 css 작업
1. 버튼크기는 한글에 맞춰 min-width로 설정합니다.
2. padding으로 좌우 여백을 줍니다.
3. 영어로 수정했을 때 레이아웃을 확인합니다.

파일/폴더/이미지 네이밍 규칙

파일명 네이밍 규칙

파일명은 두단어 이상 조합은 피합니다. 두단어 이상 조합의 경우에는 단어사이에 언더바(_)를 사용합니다. web 파일이 아닌 경우에는 파일명 시작은 서비스 타입으로 시작합니다. 서비스타입을 넣는 경우 세단어 조합까지 가능합니다. ex) user_list.html (O) / user_tpye_list.html (X) / agent_user_list.html (O) / agent_login.html (O)

폴더명 네이밍 규칙

폴더명은 축약어를 사용하지 않습니다. img, js는 예외로 축약어를 사용합니다. 업체별 폴더를 생성할 경우 폴더명 가장 마지막에 '_업체명'을 붙입니다. ex)타임키퍼_경기대 (O) / 경기대 (X)

이미지명 네이밍 규칙

파비콘과 자사로고는 favicon, logo로 합니다. 이미지 활용 타입을 분류하여 작성합니다. 클릭해서 활성화 되는 이미지는 파일명 끝에 _active를 붙입니다. 이미지의 성격에 맞춰 네이밍을 작성합니다. 축약어의 사용은 되도록 피하지만, 너무 길어지는 경우에는 3글자 또는 4글자 안으로 축약합니다. 너무 짧은 축약은 피합니다. 세 단어 조합까지만 사용합니다. 네 단어 조합은 _active를 붙였을 경우에만 가능합니다. 중복 활용의 이미지가 있는 경우에는 _숫자를 붙여줍니다. 화살표의 경우 prev와 next를 한 세트로 넘버링 합니다. *이미지 활용 타입 버튼태그 btn_ 아이콘 i_ 화살표 이전 arr_prev_ 화살표 다음 arr_next_ 일반사진 img_ 배경사진 bg_ 로고 logo_ ex) arr_prev_01.png (O) / arr_next_active_01.png (O) / btn_excel_download_check.png (X)
