HTML 작업 규칙

문서형 선언

DTD(Document Type Definition) 선언은 HTML5에서 제공하는 것을 기본으로 한다.

HTML 언어 설정

HTML 언어는 ko를 기본으로 한다.

head 작성

head는 title > meta > meta-og > favicon > css 순으로 작성한다.

script는 head 안에 넣지 않는다.

meta-tag 작성

메타태그는 charset > viewport > http-equiv > title > description > keywords > author > meta-og 순으로 작성한다.

meta-og의 경우 홈페이지에서만 사용한다.(솔루션에서는 사용하지 않는다.)

meta-og는 og:title > og:url > og:stie_name > og:description > og:imgage 순으로 작성한다.

stylesheet 순서

head에 선언되는 css의 순서는 vender > common 순으로 작성한다.

common css의 순서는 font.css > reset.css > common.css > 해당 프로젝트 css 순으로 작성한다.

파일 경로 작성

파일 경로는 상대경로로 작성한다.

파일의 위치가 같은 경우라도 기호(./)는 생략하지 않는다.

위치가 같을 때 : ./

위치가 상위에 있을 때 : ../

들여쓰기

코드의 가독성 및 오류 방지를 위해 코드 들여쓰기를 준수한다.

html의 형제 요소인 head와 body는 들여쓰기를 하지 않는다.

들여쓰기는 1탭을 기준으로 한다.

들여쓰기는 script 작성에도 동일하게 작성한다.

빈줄

빈 줄은 한 줄을 초과하지 않는다.

객체를 감싸고 있는 그룹을 구분하기 위해서 사용되는 빈 줄은 한 줄만 허용한다.

주석 다음 줄은 빈 줄로 두지 않는다.

줄바꿈

빈 줄이 생기지 않게 작업한다.

줄 글은 모두 한 줄로 작성한다.

줄바꿈을 하지 않는 경우

li(또는 기타 태그) 안에 a태그가 오는 경우 변경되는 데이터에 태그가 선언된 경우 화면에서 한 줄로 구현되는 경우(단 hepl_txt인 경우 줄바꿈으로 작업한다.)

띄어쓰기

마크업에서 띄어쓰기는 사용하지 않는다.

띄어쓰기 대신 줄바꿈을 사용한다.

주석

마크업에서는 개발이 필요한 부분에 대한 설명을 위해서만 사용한다.

Description의 약자인 D를 사용하여 주석의 말머리로 [D]를 달아준다.

function()에 대한 설명을 달아준다.

따옴표

따옴표는 작은 따옴표(또는 싱글따옴표)를 사용한다.

특정 태그 사용법

b 태그

특정 텍스트를 강조할 때 사용 보통 홈페이지에서 작업할 때 사용된다.

i 태그

기울기 태그로 사용하지 않는다. 기울기를 해야 할 때는 css로 선언한다. 아이콘, 수정되는 데이터에서 사용한다.

img 태그 사용 : 솔루션의 경우

해당 이미지가 스크린 리더기에 읽혀야 하는 상황일 때는 img 태그를 활용해서 이미지를 삽입한다. background-size:cover를 사용해야 할 경우에는 이미지를 감쌀 태그를 하나 선언하고 텍스트로 이미지에 대한 설명을 남긴다. 텍스트는 text-indent를 활용해 숨김 처리 한다.

img 태그 사용 : 홈페이지의 경우

해당 이미지가 스크린 리더기에 읽혀야 하는 상황일 때는 img 태그를 활용해서 이미지를 삽입한다. background-size:cover를 사용해야 할 경우에는 이미지를 감쌀 태그를 하나 선언하고 텍스트로 이미지에 대한 설명을 남긴다. 텍스트는 text-indent를 활용해 숨김 처리 한다.

버튼

button 태그

모든 버튼은 button type="button"을 사용한다.

a 태그

페이지 이동, 파일 다운로드를 해야하는 경우에 사용한다.

input 태그를 사용하여 버튼을 만드는 것은 지양한다.

#none 사용법

a태그의 href, option의 value에서 공란으로 처리해야하는 경우에는 #none을 사용한다.

CSS 선언

style로 CSS를 선언하는 것은 지양한다.

CSS는 CSS파일에서만 작업한다.

display:none/display:block과 같이 단순 숨김/보임 정도의 선(개발에서 작업하는 용도)만 사용한다.

form 태그

form은 개발단에서 요구하지 않은 이상 사용하지 않는다.

속성 순서

title을 쓰는 경우

title은 태그의 가장 마지막에 위치한다.

a태그

href -> target -> id -> class -> title -> style

style을 사용 해야하는 경우에는 style이 가장 마지막으로 한다.
