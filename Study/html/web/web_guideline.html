웹 접근성 지침
2021.01.18 김혜인, 김수현
웹 접근성 지침 소개
WCAG W3C에서 발표한 웹 콘텐츠 접근성 지침.
KWCAG 해외 웹 표준 기술 동향을 토대로 국내 설정에 맞게 반영된 한국형 웹 콘텐츠 접근성 지침. 원칙 4개, 지침 13개, 항목 24개로 구성. (현재 2.1 버전)
1. 적절한 대체 텍스트 제공
텍스트가 아닌 콘텐츠는 그 의미나 용도를 인식할 수 있도록 대체 텍스트를 제공해야합니다.
- alt로 대체 텍스트 제공
→ img 요소뿐만 아니라 input의 type이 image,button인 경우, 이미지맵의 area에도 대체 텍스트가 존재할 경우 alt로 정보 제공
- 마크업으로 대체 텍스트 제공
→ 대체 텍스트가 너무 긴 경우 img 태그의 alt를 빈값으로 제공하고 대체 텍스트를 마크업으로 제공하는 방법입니다. 주의할 점은 해당 마크업을 보이지 않게 처리할 때, 'display:none' 또는 'visibility:hidden'으로 처리하면 스크린리더에서 읽어주지 않기 때문에 다른 방법으로 보이지 않게 처리해야합니다.
- 데이터 차트, 복잡한 콘텐츠
→ 데이터 차트나 복잡한 콘텐츠도 사용자가 해당 콘텐츠의 의미를 충분히 파악할 수 있도록 대체 텍스트를 제공해야 합니다.
2. 자막 제공
멀티미디어 콘텐츠에 포함된 음성 및 대사와 동등하게 대체 수단을 제공해야합니다. 대체 수단은 자막, 대본, 원고, 수화 중 하나로 제공하면 됩니다.
- 제 3자가 영상을 업로드 하는 경우, 사용자가 대체 콘텐츠를 제공할 수 있도록 툴을 제공해야합니다.
- 소리는 나오지 않고 텍스트만 나오는 영상일 경우, 따로 원고를 제공하거나 하는 등의 방법으로 대체 콘텐츠를 제공해야합니다.
3. 색에 무관한 콘텐츠 인식
색을 구분하지 못하더라도 콘텐츠를 인식하는데 문제가 없어야 합니다.
- 차트 : 색으로만 구분하는 것이 아니라 패턴이나 굵기 또는 모양이나 테두리 등으로 구분 해주어야 합니다.
- 슬라이드 버튼 : 선택된 버튼을 모양으로 구분하면 명확하게 인식할 수 있습니다.
- pagination : 테두리나 굵기 또는 다른 방법으로 구분이 가능하도록 구현해야 합니다.
4. 명확한 지시 사항 제공
* 오류사례
- 모양으로만 정보 제공
- 위치나 방향으로만 정보 제공
- 크기로만 정보 제공
- 색으로만 정보 제공
5. 텍스트 콘텐츠의 명도 대비
텍스트와 배경색 간의 명도 대비는 4.5:1 이상이 되도록 구현해야 합니다.
6. 자동 재생 금지
페이지 진입 시 자동으로 음악이나 동영상이 재생되는 경우 스크린리더 음성과 겹치게 되면서 페이지의 내용을 인식하기 어렵습니다.
콘텐츠에 포함된 멀티미디어 파일은 정지 상태로 제공되어야 하고 사용자가 요구할 경우에만 재생 가능하도록 제공해야 합니다.
불가피하게 제공할 경우 해결 방안
- 3초 내에 정지
- ESC 키 선택 시 정지
- 소스 상 가장 먼저 제공하여 정지 기능 실행 가능하도록 구현
7. 콘텐츠 간의 구분
웹 페이지를 구성하는 모든 이웃한 콘텐츠는 시각적으로 구분되도록 제공해야 합니다.
- 테두리 , 구분선 삽입
- 서로 다른 무늬
- 배경색 간의 명도대비
- 줄 간격 및 글자 간격
- 기타 콘텐츠를 시각적으로 구분할 수 있는 방법 ex) 충분한 여백 등등
8. 키보드 사용 보장
웹 페이지에서 제공하는 모든 기능을 키보드만으로도 사용할 수 있도록 제공해야합니다.
9. 초점 이동
키보드에 의한 초점은 논리적으로 이동해야하며 시각적으로 구별할 수 있어야 합니다.
10. 조작 가능
사용자 입력 및 컨트롤은 조작 가능하도록 제공되어야 합니다.
컨트롤의 대각선 길이는 6mm 이상으로 구현, 테두리 안쪽으로 1px 이상의 여백을 두어야합니다.
11. 응답 시간 조절
시간제한이 있는 콘텐츠는 응답시간을 조절할 수 있어야 합니다.
12. 정지 기능 제공
스크롤 및 자동 갱신되는 콘텐츠는 사용하지 않는 것이 좋으며 사용 시에는 정지, 이전, 다음 기능을 제공해야 합니다.
13. 깜빡임과 번쩍임 사용 제한
- 초당 3~50회 주기로 깜빡이거나 번쩍이는 콘텐츠를 제공하지 않아야 한다. - 이런 컨텐츠를 제공할때는 사전에 경고하거나 중단할 수 있는 수단을 제공해야 한다. - PEAT라는 프로그램으로 확인할 수 있다.
14. 반복 영역 건너뛰기
- 컨텐츠의 반복되는 영역은 건너뛸 수 있어야 한다. - 건너뛰기 링크 제공 └ 키보드 접근 시 화면에 가장 먼저 보이게 구현 └ 마크업상 최상단에 위치 └ 링크가 페이지 내에 존재
15. 제목 제공
- 페이지, 프레임, 콘텐츠 블록에는 적절한 제목을 제공해야 한다. └ 웹 페이지의 제목은 유일하고 서로 다르게 제공해야 한다. └ 페이지 제목은 해당 내용을 이해할 수 있도록 제일 하위분류/게시판 용도에 맞게/페이지의 핵심 내용으로 적절하게 제공해야 한다. └ 프레임이나 아이프레임도 각 프레임을 설명하는 간단 명료한 제목을 제공해야 한다. └ 내용이 없는 아이프레임을 사용하는 경우에는 '빈 프레임/내용없음'등으로 제공해야 한다. └ 콘텐츠 블록에도 제목을 제공해야 한다. └ 동일한 depth의 컨텐츠는 동일한 태그로 일고나성 있게 제목을 제공해야 한다. └ 제목을 숨김 처리하는 경우는 사용하지 않아야 한다. └ 제목의 특수 기호는 하나까지만 사용을 제한한다.
16. 적절한 링크 텍스트
- 링크 텍스트는 용도나 목적을 이해할 수 있도록 제공해야 한다. └ 의미없는 빈 링크는 제거해야 한다. └ 이미지를 독립적 링크로 제공하는 경우 이미지에 alt를 잘 못 제공되거나 빈값이면 오류이기 때문에 적절한 링크 텍스트를 제공해야 한다. └ 링크 텍스트와 화면 텍스트가 중복되지 않게 작성한다.
17. 기본 언어 표시
- 주로 사용하는 언어를 명시해야 한다. - 중간에 다른 언어가 존재한다면 lang 속성으로 제공하면 된다.
18. 사용자 요구에 따른 실행
- 사용자가 의도하지 않은 기능(새창, 초점에 의한 맥략 변화 등)은 실행되지 않아야 한다. - 사용자가 예측할 수 없는 상황에서 페이지 진입 시 새 창이 열리는 경우 오류이므로 제공하지 않아야 한다. - 레이어 팝업으로 제공하는 추세이다. └ 반복 영역 건너뛰기보다 먼저 팝업을 제어할 수 있도록 구현하거나 화면을 가르지 않고 맨 윗부분에 레이어 팝업을 제공해야 한다. └ 링크 요소 내부에 블라인드로 새 창이라는 텍스트 삽입 └ 타이틀로 새 창이라고 제공 └ target="_blank"라고 적용 └ 이 세가지 방법 중 blind 클래스를 사용하여 텍스트로 안내하는 것이 가장 확실한 방법 - 사용자가 콤보 상자, 라디오 버튼, 체크 상자등을 클릭 했을 때 기능이 실행되거나 서식 제출이 일어나지 않게 작업한다. └ 닫기 버튼을 따로 제공
19. 콘텐츠의 선형 구조
- 콘텐츠는 논리적인 순서로 제공해야 한다. 사용자가 웹의 내용을 이해할 수 있도록 선형 구조로 작성되어야 한다.
20. 표의 구성
- 표는 이해하기 쉽게 구성해야 한다.
21. 레이블 제공
- 사용자 입력에는 대응하는 레이블을 제공해야 한다. - 레이블이란? └모든 사용자 입력의 용도, 역할에 대한 설명을 말한다. - 1:1 매칭의 경우 └ 레이블의 for값과 select의 id 값을 동일하게 제공 - 1:n 매칭의 경우 └ 각 입력 서식의 타이틀을 제공 - 타이틀과 레이블을 둘 다 제공한 경우에는 둘 다 스크린리더가 읽기 때문에 중복 제공을 피한다.
22. 오류 정정
- 입력 오류를 정정할 수 있는 방법을 제공해야 한다. - 입력 서식 작성 시 오류가 발생하여 입력 내용이 모두 사라지는 경우도 오류이며 작성된 내용이 삭제되지 않고 오류가 있는 부분만 수정할 수 있도록 제공해야 한다.
23. 마크업 오류 방지
- 마크업 언어의 요소는 열고 닫음, 중첩 단계 및 속성 선언에 오류가 없어야 한다.
24. 웹 애플리케이션 접근성 준수
- 콘텐츠에 포함된 웹 애플리케이션은 접근성이 있어야 한다.
